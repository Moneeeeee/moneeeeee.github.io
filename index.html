STM32F4时钟问题

发表于 2024-02-29

STM32F4时钟问题

  • 在配置USART时，baud配置为115200，通信乱码。经过测试，实际为38400.

  • TIM的频率和周期明显不符合设想配置

一般这种就是时钟有问题，通过

打印出时钟频率为

SYSCLK : 53760000

HCLK_Frequency : 53760000

PCLK1_Frequency : 13440000

PCLK2_Frequency : 26880000

很明显与默认的168M不符

经过查询和翻库，得到解决办法：

  • 文件：stm32f4xx.h

  • #define HSE_VALUE    ((uint32_t)25000000) /*!< Value of the External oscillator in Hz */

-------->

#define HSE_VALUE    ((uint32_t)8000000) /*!< Value of the External oscillator in Hz */

STM32F4 Clock Issue

  • When configuring USART, the baud rate is set to 115200, but communication results in garbled data. Upon testing, the actual baud rate is found to be 38400.

  • The frequency and period of TIM are significantly different from the intended configuration.

Usually, such issues arise due to clock problems. By using the following code snippet:

Clock frequencies are printed as:

SYSCLK : 53760000

HCLK_Frequency : 53760000

PCLK1_Frequency : 13440000

PCLK2_Frequency : 26880000

It is evident that they do not match the default 168MHz.

After research and consulting the library, the solution is obtained:

  • File: stm32f4xx.h

  • #define HSE_VALUE    ((uint32_t)25000000) /*!< Value of the External oscillator in Hz */

-------->

#define HSE_VALUE    ((uint32_t)8000000) /*!< Value of the External oscillator in Hz */

HowtoCreateAPages

发表于 2024-02-29 分类于 Help

  • 新建文章：

    hexo new <title>

    /source/_posts`目录下创建一篇新的文章

    接下来在这篇文章里使用 MarkDown 语法编写文章即可

    • Front-matter

      使用---包括起来的内容称之为Front-matte

配置项 意义
title 网页文章标题
date 文章创建如期
comments 文章评论功能是否启动
tags 文章标签
categories 文章分类
keywords 文章关键字

  • 清除掉旧的数据:

    hexo clean

    清除掉之前生成的网页，即站点根目录下的public文件夹

  • 生成新的页面：

    hexo g

    将source文件夹下所有的md文件进行渲染，生成HTML页面，存放在public文件夹下

  • 部署到云端:

    hexo d

    发布到moneeeeee.github.io

  • 上传图片:直接粘贴图片,然后使用<C-f>快捷键，将所有的文章名/替换为空即可删除

    然后再将博客上传，图片就会随着文章一起打包。在网页中就可以看到正常显示的图片，大功告成。
